# WSS Docker test stack

Runs the full WSS system locally — server and a test client — using Docker Compose.

## Prerequisites

- Docker 24+
- Docker Compose — either the plugin (`docker compose`) or the standalone binary (`docker-compose`); `test.sh` auto-detects which one is available
- `openssl` (for certificate generation)

## Quick start

The easiest path is the automated end-to-end script:

```bash
cd wss/docker
./test.sh
```

It generates certs, builds images, starts the server, registers and enrolls a
client, starts the persistent connection, and verifies both NNG management
sockets (client and server) before exiting cleanly.

### Manual steps

```bash
cd wss/docker

# 1. Generate TLS certificates (server CN: access-manager.kanyuka.info)
./gen_certs.sh

# 2. Build images
docker build -f Dockerfile.server -t wss-server:dev ..
docker build -f Dockerfile.client -t wss-client:dev ..

# 3. Start the server (memcached runs inside the server container)
docker-compose up -d server

# 4. Register a test client and capture its UUID
CLIENT_ID=$(docker-compose exec -T server \
    python /app/scripts/register_client.py --db /data/wss.db --days 1)
echo "Client ID: $CLIENT_ID"

# 5. Enroll the client (exchanges UUID for a signed certificate)
docker-compose --profile client run --rm \
    -e WSS_CLIENT_ID="$CLIENT_ID" client

# 6. Start the persistent connection (detached)
docker-compose --profile client up -d client
```

## Inspecting a running client

The client exposes an NNG management socket on port 8767. While the client
container is running, exec into it to use `wss-client-mgr`:

```bash
docker exec -it docker-client-1 python -m client_manager.main
```

Available commands:

| Command | Description |
|---|---|
| `/client ping` | Check if the management socket is reachable |
| `/client status` | Connection state, server URI, last ping/pong, uptime |
| `/client cert` | Certificate subject, validity window, fingerprint |

Or query the NNG socket directly without the interactive shell:

```bash
docker exec docker-client-1 python3 -c "
import json, pynng
with pynng.Req0(dial='tcp://127.0.0.1:8767', recv_timeout=3000, send_timeout=3000) as s:
    s.send(json.dumps({'cmd': 'status'}).encode())
    print(json.dumps(json.loads(s.recv()), indent=2))
"
```

## Inspecting the database

The SQLite database is stored on the host at `wss/docker/data/wss.db` and
bind-mounted into the server container at `/data/wss.db`. You can query it
directly without exec-ing into the container:

```bash
sqlite3 wss/docker/data/wss.db "SELECT id, allow_to FROM clients;"
```

## Regenerating certificates / full reset

The server certificate is always regenerated by `gen_certs.sh`. To force a
full reset (including the CA and all client data):

```bash
rm -rf certs/
./gen_certs.sh
rm -rf data/           # wipe server DB (host bind mount)
docker-compose down -v # wipe client_data volume so enrollment runs again
```

## Architecture

| Service  | Image         | Purpose                                                          |
|----------|---------------|------------------------------------------------------------------|
| `server` | built locally | WSS server on port 8765 (TLS, mTLS); memcached on :11211 (internal) |
| `client` | built locally | Test client (profile: `client`)                                  |

The server is reachable inside the Docker network as
`access-manager.kanyuka.info` (network alias). The client resolves this
hostname through Docker's internal DNS.

Memcached runs **inside the server container** bound to `127.0.0.1:11211` —
it is not a separate service and is not reachable from other containers.

## Environment variables

Copy `.env.example` to `.env` to customise:

| Variable               | Default                        | Description                               |
|------------------------|--------------------------------|-------------------------------------------|
| `WSS_EXTERNAL_HOST`    | `access-manager.kanyuka.info`  | Public hostname used for TLS cert CN, SSH tunnel target, and client connection. **Note:** the Docker network alias in `docker-compose.yml` is not variable-substitutable — if you change this value you must also update the `aliases` entry manually. |
| `WSS_PORT`             | `8765`                         | Host port mapped to the WSS server        |
| `WSS_MGMT_PORT`        | `8766`                         | Host port mapped to the server NNG socket |
| `WSS_CLIENT_MGMT_PORT` | `8767`                         | Host port mapped to the client NNG socket |

## Stopping

```bash
docker-compose down        # stop, keep volumes and host data/
docker-compose down -v     # stop and delete client_data volume
```
